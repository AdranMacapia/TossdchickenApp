## appSpec

### name
Toss D' Chicken

### description
Point-of-sale, recipe costing, ingredient inventory, expense tracking, and P&L reporting system for a fried chicken quick-service restaurant. Runs as a PWA installable on mobile and desktop from a single React/Vite codebase.

### auth
- Provider: Supabase Auth (email + password)
- Roles stored in `auth.users` metadata (`user_metadata.role`: `"owner"` | `"cashier"`)
- Auth seam: `src/context/AuthContext.tsx` wraps the whole app; exposes `user`, `role`, `signIn`, `signOut`
- Route guard: `<PrivateRoute role="owner">` blocks cashier from all non-POS pages

### designSystem
- **Palette:** Deep orange primary (`#E8610A`), warm white background (`#FFF8F2`), charcoal text (`#1A1A1A`)
- **Typography:** Inter (system fallback), 14px body, 12px label, 20px heading
- **Radius:** 8px cards, 4px inputs, 24px primary buttons
- **Spacing:** 4px base unit, 16px page padding on mobile
- **Tailwind config:** extend `colors.brand` with `{ primary: '#E8610A', bg: '#FFF8F2' }`
- **Tone:** Functional, warm, fast — no decorative chrome; every tap is productive

### blocks (reusable UI patterns)

- **NumericKeypad** — big-button qty input for POS, no keyboard required on mobile
- **CategoryTabBar** — horizontal scrollable tabs to filter menu items on order screen
- **PriceTag** — `₱ X,XXX.XX` formatted badge, color-coded below-margin warning
- **StockBadge** — green/yellow/red indicator for ingredient stock vs threshold
- **ReceiptLayout** — 58mm thermal-paper-like column layout for print/share

## mockData

> Seed data is derived from the actual Toss D' Chicken menu (images provided 2026-06-18).
> The app is CMS-driven: all menu changes happen in the owner's Menu Management screens
> and are immediately reflected in the POS — no code changes needed.

### Pricing model (important for POS implementation)
- **Flavor tiers**, not individual flavor prices:
  - Original = base price (no surcharge)
  - Any other flavor (Honey Garlic, Salted Egg, Buffalo, Sweet Chili, Garlic Parmesan, Cheesy Jalapeño) = +₱10
  - Jack Daniel's = +₱20
- Extras (Rice add-on, Dipping Sauce, Cajun Fries) are separate standalone menu items, not flavor surcharges
- **Barkada Feast Box** allows up to 2 flavor selections at a flat price — implement in Phase 2 as `max_flavors` field on `menu_items` (default 1)
- **"Drizzled or not"** applies to ALL flavored items (any item with a non-Original flavor). If the customer chooses "not drizzled", the sauce is served in a separate tub which costs ₱1 extra in COGS. Price to customer does NOT change. Implement as `is_drizzled boolean default true` on `order_items`; when false, add ₱1 to `unit_cost` at order completion.

## Phase Plan (14 Days)

### Phase 1 — Days 1–2: Scaffold + Auth
**Goal:** Running Vite PWA with Supabase auth, role-based routing, and DB schema live in Supabase.

Deliverables:
- `npm create vite@latest tossdchicken -- --template react-ts`
- Install: `@supabase/supabase-js react-router-dom tailwindcss vite-plugin-pwa`
- Tailwind configured with brand colors
- Supabase project created (user action); `.env.local` with `VITE_SUPABASE_URL` + `VITE_SUPABASE_ANON_KEY`
- Full DB schema SQL run in Supabase SQL editor (from HANDOFF.md)
- Seed SQL run (from mockData above)
- `src/lib/supabase.ts` — Supabase client
- `src/context/AuthContext.tsx` — session + role
- Login page (`/login`)
- Register page (`/register`)
- `<PrivateRoute>` guard component
- Role-based router: owner → `/reports/dashboard`, cashier → `/pos`
- Basic layout shell (sidebar nav for owner)

### Phase 2 — Days 3–4: POS Order Screen
**Goal:** Cashier can build an order, pick flavors, see running total, and complete a sale.

Deliverables:
- `src/pages/pos/OrderScreen.tsx`
- `src/pages/pos/Receipt.tsx`
- `src/context/CartContext.tsx` — order line items in memory
- `src/lib/costing.ts` — `calcUnitCost(item, flavor, recipes, ingredients)`
- `src/components/NumericKeypad.tsx`
- `src/components/CategoryTabBar.tsx`
- On order complete: insert `orders` + `order_items`, deduct stock from `ingredients`, write `inventory_log`
- Receipt page: printable layout, "New Order" button

### Phase 3 — Days 5–6: Menu Management
**Goal:** Owner can add/edit/delete categories, items, and flavor variants.

Deliverables:
- `src/pages/menu/Categories.tsx` — CRUD list
- `src/pages/menu/MenuItems.tsx` — CRUD list with category filter
- `src/pages/menu/Flavors.tsx` — per-item flavor CRUD
- Availability toggle (is_available) on each item
- `<PriceTag>` component with below-margin warning (uses `calcUnitCost`)

### Phase 4 — Days 7–8: Costing & Recipe Editor
**Goal:** Owner can see cost breakdown per item/flavor and edit ingredient recipes.

Deliverables:
- `src/pages/inventory/RecipeEditor.tsx` — add/remove ingredients + qty per item/flavor
- `src/pages/costing/CostingSheet.tsx` — table: item | flavor | cost | price | margin% | status
- Unit conversion logic in `src/lib/costing.ts`: purchase price → cost per usage unit
- Suggested price calculator: `price = cost / (1 - marginTarget)`
- Flag items below target margin

### Phase 5 — Days 9–10: Inventory
**Goal:** Owner can manage ingredient stock, adjust manually, and see low-stock alerts.

Deliverables:
- `src/pages/inventory/IngredientList.tsx` — full CRUD, stock level, `<StockBadge>`
- Manual stock adjustment form → writes to `inventory_log`
- Low stock summary widget (reused on Dashboard)

### Phase 6 — Days 11–12: Expenses + Reports
**Goal:** Owner can log daily expenses and view a P&L summary.

Deliverables:
- `src/pages/expenses/ExpenseLog.tsx` — add/edit/delete by date and category
- `src/pages/reports/Dashboard.tsx` — today's sales, COGS, gross profit, top items
- `src/pages/reports/PnL.tsx` — date-range picker; Sales − COGS = Gross; Gross − Opex = Net
- All report data via Supabase aggregate queries (no local computation on large datasets)

### Phase 7 — Days 13–14: PWA Polish + Settings + Deploy
**Goal:** App installs on mobile, works offline for cached views, and is live on Vercel.

Deliverables:
- `public/manifest.json` — name, icons (512×512, 192×192), theme color `#E8610A`
- `vite.config.ts` — vite-plugin-pwa with Workbox `staleWhileRevalidate` for static assets
- `src/pages/Settings.tsx` — business name, receipt header, target margin %, VAT toggle
- Lighthouse PWA audit ≥ 90
- `vercel.json` with SPA rewrite rule
- Deploy via `vercel --prod`
- Smoke test: install on Android Chrome home screen, complete one order end-to-end
